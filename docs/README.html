PeopleChain Android Web

PeopleChain is an on-device, peer-to-peer ledger designed for decentralized chat and lightweight data exchange. It runs entirely on end-user devices with no required backend, and provides a friendly Flutter SDK.

1. Concept & Purpose

Provide decentralized chat and data persistence without servers. Store data locally, exchange over encrypted data channels, and converge nodes with a simple deterministic chain rule.

2. Architecture & Workflow

  • SDK facade: PeopleChainNode for app integration
  • Crypto & keys: ed25519/x25519 from a master seed in secure storage
  • Storage: Isar on Android; in-memory mode on Web
  • Transport: WebRTC data channel; discovery via P2P manager
  • Sync: handshake + delta blocks/txs

See docs/Overview.md and Mermaid diagrams under docs/diagrams.

3. Election & Consensus

  • Prefer higher height
  • Tie-break by lexicographically larger blockId
  • Validate prev linkage and recomputed tx merkle root
  • Optional ConsensusHook for app-specific validation

4. Synchronization & Propagation

  • Exchange hello/hello_ack with tip info
  • Request missing ranges via get_blocks; fetch missing tx via get_txs
  • Announce new items with inv_txs and inv_blocks

5. Integration Options

  • Embed the Flutter SDK (peoplechain_core) in your app
  • On Web, toggle the API tab in the Testing Dashboard to expose a postMessage API

7. Getting Started with PeopleChain (Production)

Web deployment

  1. In Dreamflow, click Publish and choose Web. This builds and deploys the current project. Branding is already updated in web/.
  2. Alternatively, download the code and run flutter build web --release. Host build/web on your static hosting.

Android release

  1. In Dreamflow, use Publish → Google Play for store deployment.
  2. Or locally: set your applicationId in android/app/build.gradle, configure signing, then run flutter build appbundle --release.

Connecting live nodes

  1. Open the app on two devices/browsers. From the home screen’s WebRTC panel, click Create Offer on device A.
  2. On device B, paste the Offer into Accept Offer. Copy the generated Answer back to A and use Accept Answer.
  3. Once the channel shows Open, exchange messages. Open the beaker icon to inspect Peers, Chain, and Explorer in real time.

Notes about optional backends

PeopleChain requires no backend. If you later want Firebase or Supabase, use Dreamflow’s Firebase or Supabase panel to connect a project and follow the guided setup. Avoid CLI configuration inside Dreamflow.
